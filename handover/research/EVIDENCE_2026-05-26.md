## 3. Round 1 ŌĆö the deliberate-by-flaw test failure (preserved for audit)

First run revealed 1 FAIL:

```text
---- market_unknown_side_is_unknown_variant stdout ----
thread 'market_unknown_side_is_unknown_variant' panicked at tests/robustness.rs:169:5:
assertion `left == right` failed
  left: EnvelopeFieldTooLarge
 right: EnvelopeUnknownVariant
```

Root cause: `side` was given a 4-byte hard cap; `"MAYBE"` (5 bytes) hit the
size check before the enum-discriminator match.

Fix: 3 fields (`final_answer_letter`, `side`, `directive_kind`) had their
narrow byte caps replaced with `MAX_FIELD_BYTES` (2KB). Discriminator
checks are now done by the enum match, not by a tight size limit.

This is an important research finding: **schema-level "size cap" and
"enum discriminator" are two distinct gates and must be ordered correctly**.
The Phase A implementation must preserve this ordering.
