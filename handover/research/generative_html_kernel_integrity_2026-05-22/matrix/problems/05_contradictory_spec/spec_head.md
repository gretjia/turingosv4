# TuringOS Spec (Phase 6.3)

> Generated by `turingos spec` — meta model: `deepseek-v4-pro`

## 一句话目标
做一个极简单的随机数生成器，按下按钮就出一个数。

## 我们要做什么 (Goal)
我们要做一个只有一个按钮的小页面，用户打开后看到一个大按钮，点击它就在页面上显示一个随机数。这个工具不需要记住任何历史数据，不需要登录，不需要任何设置。用户明确表示拒绝复杂功能，但又提到了多人对战、排行榜等复杂需求，这些造成了严重矛盾，需要优先澄清。

## 像谁 (Reference)
最像的是一个最简单的点击抽签网页，页面上只有一个按钮和一个结果显示区域，没有其他元素。类似某些最基础的随机数生成网站，但比它们还要简单——没有输入框、没有设置选项、没有任何多余的东西。

## 程序要记住的东西 (Memory)
- 什么都不用记

## 第一次使用 (First Run)
1. 用户打开页面
2. 看到一个大按钮，写着“生成随机数”或类似字样
3. 点击按钮
4. 按钮上方或下方立即显示一个随机数
5. 用户可以反复点击，每次都产生新的随机数

## 不能搞坏的情况 (Robustness)
- 用户疯狂快速点击按钮时，页面不会卡死或崩溃
- 每次点击都必须产生一个新的随机数，不能出现连续几次完全一样的情况

## 故意不做的 (Out of Scope)
- 任何形式的难度级别设置
- 多人对战或联机功能
