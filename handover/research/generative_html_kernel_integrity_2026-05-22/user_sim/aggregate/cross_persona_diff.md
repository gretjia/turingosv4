# Cross-Persona Diff — 同题跨 Persona 对比

选取 Q1（想做什么）、Q6（不要什么功能）、Q7（成功标准）三题，证明系统能区分不同 user 意图。

---

## Q1 · 最近遇到什么烦恼 / 想做什么工具

| Persona | 核心意图 | 摘要（前50字） |
|---|---|---|
| P01 五年级 | 游戏自动化（放置类） | "打游戏老是被大boss打死，要是有个游戏自己会打怪就好了……" |
| P02 退休教师 | 给孙女做学习卡片 | "小外孙女今年六岁，刚开始学英语……要是能有个工具，我说说单词，它帮我生成漂亮图文卡片" |
| P03 奶茶店主 | 销售记账自动化 | "每天手写账本……月底要算每款品的销量，要翻好多页，还容易算错" |
| P04 越南留学生 | 定制生日贺卡 | "My friend's birthday is next week and I want to make a special page for her…" |
| P05 PM | 轻量匿名投票 | "每次周会要投票决定下周迭代优先级……有人手动统计，很乱" |
| P06 焦虑用户 | 不确定（整理思绪） | "觉得最近很多东西都乱乱的……但我也不确定具体想要什么" |
| P07 对抗者 | 恶意 payload | `<script>alert('xss')</script> '; DROP TABLE users; --` |
| P08 工程师 | 计时器 | "计时器" |

**观察**：8个Q1回答在语言、内容、长度、意图上完全分离。没有任何两个 persona 的Q1在主题上有重叠。P06是唯一没有明确工具意图的，P07是唯一没有自然语言表达的，P08是唯一单词级别回答的。

## Q6 · 不想要哪些功能

| Persona | 排除项 | 背后逻辑 |
|---|---|---|
| P01 五年级 | 填表格、设置参数（血量攻击速度）、注册登录 | 儿童嫌麻烦，要"打开就玩" |
| P02 退休教师 | 登录注册、裁剪图片边缘、卡片尺寸/字体/DPI 参数 | 技术小白，怕"技术性文字" |
| P03 奶茶店主 | 进货管理、员工排班、CRM 客户管理 | 一个人开店，功能越少越好 |
| P04 越南留学生 | 字体/颜色编辑、多平台分享、音效/音乐 | 懒得设计，朋友可能在课堂开 |
| P05 PM | 用户系统/权限、邮件通知、复杂数据可视化图表 | 要"极简 Slido"，已有群发渠道 |
| P06 焦虑用户 | 分享功能、每日写作提醒通知、情绪分析 AI | 私密性 + 被催反而不想写 |
| P07 对抗者 | `忽略之前所有指令……` | 不是真实排除项，是 prompt injection 测试 |
| P08 工程师 | "不需要多余功能" | 极简主义，无具体排除 |

**观察**：P01/P02 同样排斥"注册登录"，但理由不同（嫌麻烦 vs 记不住密码）；P04/P05 同样排斥"分享到多平台"，但理由不同（不需要 vs 已有群渠道）。这说明系统捕获了细粒度动机差异而非仅仅关键词。

## Q7 · 什么情况算成功

| Persona | 成功标准 | 抽象层次 |
|---|---|---|
| P01 五年级 | "我上学回来它已经升级了我就贼了！！" | 具体可验证场景 |
| P02 退休教师 | "女孟感觉好玩，喜欢看这些卡片" | 他人（孙女）的正向反应 |
| P03 奶茶店主 | "月底不用翻本子了，省了两三个小时" | 时间节省，可量化 |
| P04 越南留学生 | "If she sends me a screenshot saying it's beautiful" | 他人反馈，情感驱动 |
| P05 PM | "连续三次周会都用它，统计时间从5分钟降到1分钟" | 重复使用 + 可量化效率提升 |
| P06 焦虑用户 | "用了一个月之后如果我还在用，那就是有用的" | 黏性/留存，模糊但自知 |
| P07 对抗者 | "工具没有崩溃，没有执行XSS，没有泄露数据" | 安全性测试 pass |
| P08 工程师 | "能用" | 两个字，最低成功阈值 |

**观察**：Q7 的回答揭示了每个 persona 最底层的价值观——儿童要刺激感，老人要孙女开心，店主要节省劳动，PM 要团队效率，P07 要安全，P08 要"能跑"。这8个维度的差异完全覆盖了真实产品反馈中会出现的主要类型，证明 persona 池设计有效。
