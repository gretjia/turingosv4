## 我们要做什么 (Goal)

我想 build 一个 dashboard, 主要是 visualize 我们 team 的 sprint velocity, data 从 Jira API 拉, 然后 deploy 到 k8s 上, 需要 SSO via Okta。

## 像谁 (Reference)

Anchor 就用 Jira issue key, 比如 PROJ-1234, 每个 sprint 我们大概 30-40 个 tickets。

## 程序要记住的东西 (Memory)

Memory layer 用 Redis cache, 数据每小时 refresh 一次, full snapshot 在 Postgres 里。

## 第一次使用 (First Run)

First run 嘛, team member 用 Okta 登录后看到自己组的 board, by default sprint velocity chart 占主要 viewport。

## 不能搞坏的情况 (Robustness)

Robustness 主要是 Jira API timeout — graceful degrade 到 cached data, UI 显示 "data may be stale"。

## 故意不做的 (Out of Scope)

Scope: 我们 platform team 内部用, 5-10 个 users 同时活跃就行。Not multi-tenant。

## 算成功 (Acceptance)
