# TuringOS Spec (Phase 6.3)

> Generated by `turingos spec` — meta model: `deepseek-v4-pro`

## One-line Goal
A single self-contained browser page that helps me compare three project launch plans by scoring them and seeing which one wins.

## What We're Building (Goal)
A simple decision tool that lives in one HTML file. I open it in my browser, see three launch plans side by side, give each plan a score from 1 to 5 on a few criteria, and the page instantly shows me weighted totals and a clear recommendation. Nothing gets saved to a server or account — it all stays in the page while I'm using it.

## Like What (Reference)
A lightweight decision matrix app, but stripped down to the absolute minimum: no accounts, no databases, no external services. Think of a paper pros-and-cons list, but with automatic math.

## What the Program Remembers (Memory)
- Three launch plan names
- Criteria names and their importance weights
- Scores (1–5) for each plan on each criterion
- Weighted totals and the recommended plan
- All of the above only while the page is open

## First Run (First Click Walk)
1. I open the single HTML file in my browser
2. I see three plan columns with empty score slots
3. I type in my plan names and criteria
4. I set how important each criterion is (the weight)
5. I click or type a score from 1 to 5 in each slot
6. The weighted totals update immediately as I change scores
7. A highlighted recommendation appears showing which plan scored highest

## What It Must Not Break On (Robustness)
- Blank or missing scores should not crash the page
- Typing letters or symbols into a score field should be handled gracefully
- All three plans having equal totals should still show a sensible result
- Very long plan names should not break the layout
- The page must work completely offline with no internet connection

## Deliberately NOT Doing (Out of Scope)
- No login or user accounts
- No backend server or database
- No cloud sync or saving across devices
- No collaboration or sharing features
- No payments or subscriptions
- No external packages, libraries, or frameworks
- No remote fonts or CDN resources
- No external URLs of any kind

## Success Looks Like (Acceptance)
- I can paste a launch scenario into the page and compare options in under five minutes
- I can explain my final choice to someone else using what the page shows me
- The page never needs the internet and never calls home

## Given/When/Then Examples
- **Given** I have three plans with different scores, **When** I change one score from 3 to 5, **Then** the weighted total and recommendation update immediately
- **Given** I leave a score blank, **When** I look at the totals, **Then** the page treats it as zero and still shows results for the other scores
- **Given** I type "hello" into a score field, **When** I move to the next field, **Then** the page either clears it or ignores it without breaking
- **Given** all three plans end up with the same weighted total, **When** I view the recommendation, **Then** the page tells me it's a tie rather than picking one at random
- **Given** I close the browser tab and reopen the file, **When** I look at the page, **Then** it starts fresh with empty scores (no memory of my last session)

## One-line Brief to the AI Coder
Build a single self-contained HTML file with a three-column decision matrix where I can name plans, set criteria weights, enter 1–5 scores, and instantly see weighted totals and a highlighted recommendation — no external dependencies, no internet required, and it must handle blank or non-number input without breaking.
