# TuringOS Spec (Phase 6.3)

> Generated by `turingos spec` — meta model: `deepseek-v4-pro`

## One-line Goal
A single self-contained web page that lets me compare three project launch plans side-by-side, score them, and instantly see which one wins.

## What We're Building (Goal)
A lightweight decision tool that runs entirely in the browser. I open one page, see three plan columns, give each plan a score from 1 to 5 against weighted criteria, and immediately see the weighted totals and a clear recommendation. Nothing is saved to a server, no account is needed, and everything resets when I close the page.

## Like What (Reference)
A simple decision matrix app — but stripped down to just one page, no accounts, no databases, no external services.

## What the Program Remembers (Memory)
- Three plan names and their scores
- The criteria and how important each one is (weights)
- The current score table while the page is open
- Which plan is currently recommended

## First Run (First Click Walk)
1. Open the page and see three empty plan columns side by side
2. See a list of criteria with weight sliders or number boxes
3. Type a name at the top of each plan column
4. Click into each score cell and type a number from 1 to 5
5. Watch the weighted totals update immediately at the bottom of each column
6. See one plan highlighted as the recommended choice
7. Adjust any score or weight and see the recommendation change instantly

## What It Must Not Break On (Robustness)
- Blank or missing scores should not crash the page
- Typing letters where numbers belong should be handled gracefully
- Very long plan names should not break the layout
- All three plans having equal totals should still show a sensible result
- The page should work without any internet connection

## Deliberately NOT Doing (Out of Scope)
- No login, accounts, or user registration
- No backend server or database
- No cloud sync or saving between sessions
- No collaboration or multi-user features
- No payments or subscriptions
- No external packages, libraries, or CDN dependencies

## Success Looks Like (Acceptance)
- I can paste a launch scenario into the page and compare options in under five minutes
- After a month of use, I can explain my final choice clearly using the scores
- The page works offline and never asks me to sign up or connect to anything

## Given/When/Then Examples
- **Given** I open the page for the first time, **when** I look at the three columns, **then** I see empty score cells and no recommendation yet
- **Given** I have filled in all scores and weights, **when** I change one score from 3 to 5, **then** the weighted total and recommendation update immediately
- **Given** I type "hello" into a score cell, **when** I move to the next cell, **then** the page treats it as blank or zero instead of breaking
- **Given** all three plans end up with the same total, **when** I look at the recommendation area, **then** it tells me there is a tie rather than picking one arbitrarily
- **Given** I close the browser tab and reopen it, **when** the page loads, **then** I see a fresh empty matrix with no old data

## One-line Brief to the AI Coder
Build a single self-contained HTML file with a three-column decision matrix, editable scores (1–5), weighted criteria, instant totals, and a clear recommendation — no external dependencies, no backend, no accounts.
