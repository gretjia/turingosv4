`http://localhost:8080/build?mode=driven`.

The web path is BROKEN at HEAD 3e0fa79c: `src/web/spec.rs::build_web_turn_prompt_json`
(line ~1325) builds the messages array WITHOUT the meta-prompt content
as the first system message. cmd_spec.rs's equivalent CLI helper
`build_turn_prompt_json` (line 608-655) DOES include the meta-prompt as the
first system message. The cmd_llm `--meta-prompt` flag is informational only
(records sha256 → system_prompt_template_hash but does NOT prepend the
methodology into the LLM messages).

  {"error":"llm complete failed: llm complete returned ok=false; content=<empty>",
   "kind":"shellout_failed"}
because the LLM (DeepSeek-V3.2 via SiliconFlow) receives only a coverage-state
system message + "Produce your turn-1 output per the contract." user message,
with no rubric. It responds with markdown narrative, not the strict-JSON
envelope; cmd_llm's --strict-json gate fails and returns ok=false.

Direct verification:
